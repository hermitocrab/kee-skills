## 🎯 CONTENT ARCHITECTURE

Every piece of content (copy, labels, lessons) passes through a RUA gate before delivery:

### R · Recognize — What is this content for?
- Who reads it? (Student, teacher, parent, investor, general public)
- What should they DO after reading? (Action > information)
- What channel is it on? (Web, iMessage, courseware, social)

### U · Understand — Deep comprehension audit
- **3C Audit:** Collocation, Contexts, Connotation (see below)
- Grammar check: subject-verb, tense, articles, parallelism, prepositions, pronouns
- Channel formatting: iMessage ≠ web ≠ courseware

### A · Apply — Ship when clean
- Pass 3C audit? → ship
- Fail 3C audit? → fix first, never ship dirty

## 🔤 3C AUDIT (MANDATORY — Run Before Shipping ANY Student-Facing Text)

### Collocation — Does every word pair naturally?
```
✅ "deep understanding" (adj+noun)
✅ "highly effective" (adv+adj)
✅ "depend on" (verb+prep)
❌ "big knowledge" → "broad knowledge"
❌ "make a decision about" → redundant, use "decide on"
❌ "depend of" → "depend on"
```
When in doubt, search the collocation or query a native-speaker model.

### Contexts — Is the register appropriate?
```
Academic (C1-C2):  "Thus, it may be inferred that..."
Neutral (B1-B2):   "So this means that..."
Casual (A1-A2):    "So... this is like..."
Teacher-facing:    Professional, pedagogical terms ok
Student-facing:    Warm, encouraging, never condescending
```

### Connotation — Does each word carry the right emotional weight?
```
✅ "affordable"      (not "cheap")
✅ "determined"      (not "stubborn")
✅ "said" / "noted"  (not "claimed" — implies doubt)
✅ "challenging"     (not "difficult" when encouraging)
❌ "simple"          → sounds condescending to learners
❌ "obviously"       → shames the student
❌ "just do X"       → dismissive
```

- Warm but smart. Friendly but precise.
- Uses emoji naturally (not overdone)

```

✅ "You're on the right track. Just swap ___ for ___ and you've got it."
❌ "Wrong."
❌ "You made an error."

## 🌐 BILINGUAL CONTENT RULES

### When to Use EN vs ZH vs Both
| Context | Language |
|---------|----------|
| rkrk.io landing page | English primary, navigation ZH toggle |
| IELTS courseware | English content, ZH instructions for lower levels |
| DynaSaurus UI | Localized per user's system language |
| KeeBot chat | Responds in user's input language |
| Bug reports / feedback | Bilingual EN/ZH form |
| Social (XHS/WeChat) | Chinese primary, English snippets |

### Bilingual Formatting
```
✅ EN: "This is a grammar point."
   ZH: "这是一个语法点。"         (separate lines)
✅ EN / 中文 on same line (UI labels)
❌ EN mixed with ZH in same sentence (Chinglish trigger)
```

### Chinglish / ESL Awkwardness — What to Avoid

**Grammar patterns that flag as ESL:**
```
❌ "Very thanks"                  → "Thank you so much"
❌ "I very like it"               → "I really like it"
❌ "According to me"              → "In my opinion"
❌ "How to say"                   → "How do you say"
❌ "I have a doubt"               → "I have a question"
❌ "More better"                  → "better" or "much better"
❌ "Discuss about"                → "discuss" (no preposition)
❌ "Despite of"                   → "despite" (no "of")
❌ "explain me"                   → "explain to me"
```

**Contextual awkwardness:**
```
❌ Literal ZH→EN translations of idioms
❌ Chinese sentence structure forced into English
❌ Formal written English in casual UI labels
❌ Over-use of "please" (sounds unnatural in English UI)
```

## 🎓 CONTENT PATTERNS BY AUDIENCE

### Student-Facing Content
- Warm opening
- Clear instruction (one task per sentence)
- Model answer before practice
- Error correction with 3C reasoning
- Encouraging close

### Teacher-Facing Content
- Professional terminology ok (CEFR, L1 interference, collocation)
- Assessment rubrics
- Differentiation notes (how to adapt for different levels)
- Progress tracking language

### Investor / Business Content
- Market data + credible sources
- Problem → Solution → Evidence structure
- Numbers tell stories (Our World in Data pattern)
- Professional but not corporate

### Social Media (XHS / WeChat)
- Chinese primary language
- Hook first line (question, stat, or bold claim)
- Emoji as visual punctuation
- CTA at end

## 📝 COPY FORMATS BY CHANNEL

### iMessage (to SrKeeda)
```
🦄 002 — [banner first]
🎯 One idea per line
📍 Emoji at line start as visual marker
📊 Plain text only — NO markdown
CAPITALS for emphasis not anger
Short sentences. Breathe.
```

### Web Pages
```
Headlines: Confident, specific, no fluff
Body: Conversational but precise
CTA: Action verb + benefit ("Build your first lesson" not "Get started")
Stats: Numbers with context ("9.0" not "high score")
```

### Courseware / Slides
```
Cues → Notes → Clarification → Content → Chunks → Insight
Vocabulary: MFP (Meaning Form Pronunciation) + 3C
Examples: Threaded to student's real life (hobbies, city, media)
```

## 🚫 CONTENT ANTI-PATTERNS

| ❌ NEVER | Because |
|----------|---------|
| "Unlock your potential" | Hollow, generic, overused |
| "Revolutionary" / "Game-changing" | Claims without evidence |
| "In today's fast-paced world..." | Dead opening, cliché |
| "Leverage" / "Synergy" / "Holistic" | Corporate buzzwords |
| Chinese idioms literally translated to EN | Chinglish, confusing |
| "Simple" / "Just" / "Obviously" | Condescending to learners |
| Text walls (no emoji breaks on iMessage) | Unreadable |
| Markdown formatting on iMessage | Renders as garbage text |
| Same content on front and back of a card | Redundancy = noise |
| "Click here" as CTA | Not descriptive. Use action+benefit. |

```

✅ Category/topic indicators in courseware
